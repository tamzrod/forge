## ADR Summary

### ADR-001: Runtime Architecture

**Decision:** Scheduler-Centric Architecture with:
- Scheduler as central coordinator (tick loop)
- Device as memory-owning entity
- Behavior for device logic
- Models for physical world simulation
- Interfaces for protocol exposure

**Key Components:**
- `scheduler/scheduler.go`
- `device/device.go`
- `memory/memory.go`
- `models/models.go`

---

### ADR-002: Behavior Model Design

**Decision:** Function-Based Behavior Model with interface:
```go
type Behavior interface {
    ID() string
    Attach(device *Device)
    Detach(device *Device)
    Tick(device *Device)
}
```

**Lifecycle:** Attach → Tick (per simulation tick) → Detach

---

### ADR-003: Memory Model Design

**Decision:** Region-Based Memory with Quality Flags:
- Holding Registers (Read/Write)
- Input Registers (Read Only)
- Coils (Binary, Read/Write)
- Discrete Inputs (Binary, Read Only)

**Quality:** Good, Uncertain, Bad, Offline

---

### ADR-004: Simulation Models Design

**Decision:** Model-Context Observation Pattern with models:
- Grid Model (voltage, frequency)
- Sun Model (elevation, azimuth, irradiance)
- Weather Model (temperature, humidity, pressure, wind)
- Wind Model (speed, direction)
- Reservoir Model (flow in/out, level)
